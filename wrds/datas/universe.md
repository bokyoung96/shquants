# WRDS US Universe

This document summarizes the latest US stock universe files generated by the WRDS helper code.

The universe is stored as date intervals, not as a daily 0/1 cross-section. A row is active on a target date when:

```text
start_date <= target_date <= end_date
```

For tradable point-in-time membership, the date filter is combined with active status filters:

```text
tradingstatusflg = 'A'
conditionaltype = 'RW'
```

| File | Source | Rows | Meaning | Main Use |
| --- | --- | ---: | --- | --- |
| `names.csv` | Latest CRSP stock name history | 61,221 | CRSP security name, ticker, CUSIP, exchange, type, and status history | Debug CRSP-side identifier/name/status changes |
| `factset_links.csv` | WRDS CRSP-FactSet link table | 113,170 | CRSP `permno`/`permco` to FactSet identifier link history | Debug FactSet mappings |

| `at_20251231.csv` | `history.csv` filtered by date and active status | 3,824 | Point-in-time tradable universe at `2025-12-31` | Backtest-ready membership for that date |

## Date Fields

| Column | Source | Meaning |
| --- | --- | --- |
| `namedt` | CRSP | Start date of a CRSP name/listing row. The ticker, company name, exchange, type, status, and CUSIP fields are valid from this date. |
| `nameendt` | CRSP | End date of a CRSP name/listing row. |
| `securitybegdt` | CRSP | Security-level start date. This can span multiple name rows. |
| `securityenddt` | CRSP | Security-level end date. |
| `link_bdate` | FactSet link | Start date when the CRSP security is linked to the FactSet identifiers in that row. |
| `link_edate` | FactSet link | End date when the CRSP-to-FactSet link is valid. |
| `start_date` | Generated | Effective universe/mapping start date. Usually the later of `namedt` and `link_bdate` when a FactSet link exists. |
| `end_date` | Generated | Effective universe/mapping end date. Usually the earlier of `nameendt` and `link_edate` when a FactSet link exists. |

## How To Read The Universe

### Point-in-time membership

To get the tradable universe for one date, filter `history.csv`:

```text
start_date <= date <= end_date
tradingstatusflg = 'A'
conditionaltype = 'RW'
```

Example: if a row has `start_date=2010-01-04` and `end_date=2020-03-13`, that security is eligible through `2020-03-13` and disappears after that date unless another valid active row exists.

### Daily 0/1 membership

The current files do not store daily 0/1 membership directly.

Instead, expand intervals only when needed:

| Date | PERMNO | In Universe |
| --- | ---: | ---: |
| 2025-12-30 | 10107 | 1 |
| 2025-12-31 | 10107 | 1 |
| 2026-01-02 | 10107 | 0 |

This avoids storing a very large full matrix across all historical dates and all securities.

## Practical Rule

Use `history.csv` for backtests and point-in-time universe construction.

Use `current/universe.csv` for the latest available tradable universe.

Use `latest.csv` only for quick lookup of the latest known identifiers per `permno`.

Use `names.csv` when checking CRSP ticker, CUSIP, company-name, exchange, type, or status history.

Use `factset_links.csv` when checking FactSet identifier mappings.
